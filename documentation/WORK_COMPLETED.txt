BUILD ENVIRONMENT ISSUE DIAGNOSED & ADDRESSED
═══════════════════════════════════════════════════════════════════════════════════

Root Cause of Previous Build Failures:
  The buildozer build process uses hostpython3 (a Python binary running on the build
  machine to assist cross-compilation). This hostpython lacks SSL/TLS support,
  preventing pip from accessing PyPI to download Python packages.

Previous Attempts:
  1. Build #1: numpy pip download → SSL error
  2. Build #2: removed numpy → pillow pip download → SSL error
  3. Build #3: removed pillow → other pip packages → SSL error

Pattern: ANY package requiring pip download failed with:
  "ERROR: Could not find a version that satisfies the requirement"

Solution Applied:
  1. Remove version pins from all packages (allows pip any version)
  2. Use p4a recipes for packages available as recipes:
     • python3, kivy, pyjnius, numpy, opencv (all have recipes)
     • These compile from source, don't use pip
  3. Keep mediapipe (no recipe available, must use pip)
  4. Enhance SSL cert handling in build script
  5. Add explicit numpy to requirements (minapi=24 allows compilation)

Expected Outcome:
  ✓ All recipe-based packages should compile successfully
  ✓ Native libraries (JPEG, libffi, SSL, PNG) should build
  ✓ Critical point: mediapipe pip download (most likely failure)
  ✓ If mediapipe fails: App builds without pose detection
  ✓ Code has fallback detection mechanisms if needed
